Fluentd Open-Source Log Shipping Integration

Use industry-standard open source collectors like Fluentd to securely forward application log files to OCI Log Analytics endpoints.

Fluentd Configuration (td-agent.conf) 
<source>
  @type tail
  path /var/log/app/app.log
  pos_file /var/log/td-agent/app.log.pos
  tag oci.loganalytics.portal
  <parse>
    @type json
    time_key timestamp
    time_format %Y-%m-%dT%H:%M:%S.%L%z
  </parse>
</source>

<match oci.loganalytics.**>
  @type oci_log_analytics
  namespace <TENANCY_NAMESPACE>
  log_group_id ocid1.loggroup.oc1.eu-frankfurt-1.aaaaaaaaxf...
  log_source_name OCI_APM_Trace
  co_id ocid1.compartment.oc1..aaaaaaaasx...
  region eu-frankfurt-1
  auth_type instance_principal
</match>

Developer Checklist

  • Verify Agent State: Ensure `mgmt_agent` daemon is active and configured with a valid registration key.
  • Log Ingestion: Verify logs appear under OCI Log Analytics Trace Source using the `Search` panel.
  • Metadata Tagging: Enrich Fluentd records with `host.name` and `app.environment` fields for multi-dimensional filtering.

OCI Monitoring MQL & Prometheus Scraping

Configure OCI Monitoring alarms using Metric Query Language (MQL), or pull metrics directly into an open-source Prometheus/Grafana stack using OCI Management Agent endpoint exporters.

OCI MQL Expression 
# Trigger alert if the HTTP error rate exceeds 5% for three consecutive 1-minute intervals
HttpRequests[1m]{resourceId = "ocid1.instance.oc1..."}.status_code != "200"
| stats count() as ErrorCount
| HttpRequests[1m]{resourceId = "ocid1.instance.oc1..."}.stats(count) as TotalCount
| (ErrorCount / TotalCount) * 100 > 5
Prometheus YML Scrape block 
scrape_configs:
  - job_name: 'oci-management-agent-metrics'
    scrape_interval: 15s
    metrics_path: '/metrics'
    static_configs:

    metric_relabel_configs:
      - source_labels: [__name__]
        regex: 'oci_computeagent_(.*)'
        target_label: __name__

OpenTelemetry distributed tracing context propagation

Propagate tracing context across microservices using W3C Trace Context HTTP headers, and capture execution spans for SQL queries.

OpenTelemetry Span Tracing 
from opentelemetry import trace

tracer = trace.get_tracer("demo-portal")

def get_treasury_records(db_conn, search_term):
    # Start a span to trace this transaction
    with tracer.start_as_current_span("treasury.search") as span:
        span.set_attribute("db.system", "mssql")
        span.set_attribute("db.statement", f"SELECT * FROM treasury WHERE name = '{search_term}'")
        
        # Verify SQL injection attempt (Level 3 security attribute tracking)
        if "'" in search_term:
            span.set_attribute("security.attack.type", "sqli")
            span.set_attribute("security.attack.severity", "critical")
            
        cursor = db_conn.cursor()
        cursor.execute("SELECT * FROM treasury WHERE name = %s", (search_term,))
        return cursor.fetchall()
W3C Trace Context Propagation 
import httpx
from opentelemetry import trace
from opentelemetry.propagate import inject

def call_inventory_service():
    # Inject current span context into W3C traceparent headers
    headers = {}
    inject(headers)
    
    # headers will contain:
    # 'traceparent': '00-4bf92f3577b34da6a3ce929d0e0e4736-00f067aa0ba902b7-01'
    
    response = httpx.get("http://inventory.service/items", headers=headers)
    return response.json()

Developer Checklist

  • W3C Header Support: Verify HTTP client and proxy elements do not strip `traceparent` or `tracestate` headers.
  • Database Diagnostics: Log query execution latencies and cross-reference them with Database Performance Hub session IDs.
  • Secure Data Keys: Ensure APM private datakeys are kept secure and loaded via environment variables, not hardcoded in source.

{
  "mcpServers": {
    "oci-observability": {
      "command": "python",
      "args": ["-m", "mcp_server.main"],
      "env": {
        "OCI_COMPARTMENT_OCID": "ocid1.compartment.oc1..aaaaaa...",
        "OCI_AUTH_MODE": "instance_principal",
        "OCI_LOG_ANALYTICS_NAMESPACE": "tenancy_namespace"
      }
    }
  }
}
OCI OCL Query 
# Cluster application logs over the last 24 hours to isolate warning and error anomalies
'Log Source' = 'OCI APM Trace'
| where 'Entity Type' = 'Host'
| cluster
| where 'Cluster Type' = 'Anomaly'
| stats count() as Occurrences by 'Cluster Content'
| sort - Occurrences

Developer Checklist

  • Cluster Settings: Ensure Log Analytics entity mappings are complete so log clusters link correctly to host names.
  • MCP Permissions: Apply strict IAM policies limiting the MCP API credentials to read-only metadata calls.
  • Trend Accuracy: Operations Insights require a minimum of 14 days of metrics data before displaying capacity forecasting charts.
